### 2.1 二叉树的非递归遍历

#### 2.1.1 问题描述

已知一组 `push/pop` 操作序列，要求输出对应二叉树的后序遍历结果。

#### 2.1.2 基本要求

1. 正确解析 `2n` 条操作指令。
2. 根据指令提取前序和中序序列。
3. 输出正确的后序遍历。
4. 保证边界情况下下标和递归区间不越界。

#### 2.1.3 数据结构设计

```cpp
struct TreeNode
{
    char data;
    TreeNode *left, *right;
    TreeNode(char data) : data(data), left(NULL), right(NULL) {}
};

vector<char> pre_order;
vector<char> in_order;
```

设计说明：
1. `pre_order` 和 `in_order` 分别保存由操作序列恢复出的前序、中序。
2. `TreeNode` 仅用于重建阶段和后序输出阶段，结构简洁。

#### 2.1.4 功能说明（函数、类）

```cpp
TreeNode *BuildTree(int pre_left, int pre_right, int in_left, int in_right);
```

功能说明：
1. 在给定区间内递归重建子树。
2. 根节点取 `pre_order[pre_left]`。
3. 在中序区间定位根并切分左右子树。

```cpp
void postOrder(TreeNode *root);
```

功能说明：
1. 按“左-右-根”输出后序序列。
2. 空节点直接返回。

#### 2.1.5 调试分析（遇到的问题和解决方法）

问题与排查重点：

1. 序列关系是否提取正确。
问题现象：后序输出错误时，第一直觉容易怀疑递归函数。
排查过程：先打印 `pre_order` 和 `in_order`，确认 `push` 对应前序、`pop` 对应中序，再进入建树调试。
结论：先验证输入映射，再查建树，效率明显更高。

2. 递归区间切分是否一致。
问题现象：部分用例会少节点或重复节点。
排查过程：逐层打印 `pre_left/pre_right/in_left/in_right` 与 `left_len`。
结论：只要根在中序的位置和左子树长度一致，区间切分就稳定。

#### 2.1.6 总结和体会

这题的核心在看懂操作语义：**`push` 记录的是前序访问，`pop` 记录的是中序回退**。把这个关系看清后，后续流程就变成标准的“前中序重建 + 后序输出”。

复杂度分析：
1. 指令解析：$O(n)$。
2. 建树：当前实现每层在线性扫描中序定位根，整体为 $O(n^2)$。
3. 后序遍历：$O(n)$。
4. 总时间复杂度：$O(n^2)$，空间复杂度：$O(n)$。

### 2.2 感染二叉树需要的总时间

#### 2.2.1 问题描述

给定二叉树结构和起始感染节点，求感染扩散到所有可达节点所需分钟数。

#### 2.2.2 基本要求

1. 正确读取每个节点左右孩子编号。
2. 处理感染向下和向上（父节点）传播。
3. 按“分钟”分层计时输出结果。

#### 2.2.3 数据结构设计

```cpp
vector<int> connect[100005];
bool visited[100005];
```

设计说明：
1. `connect` 把树转成无向图邻接表，支持双向传播。
2. `visited` 防止重复入队和死循环。

关键建图片段（原注释保持不变）：

```cpp
if (lc != -1)
{
    connect[i].push_back(lc); // i 可以感染 lc
    connect[lc].push_back(i); // lc 也可以感染 i (逆向传播)
}
```

#### 2.2.4 功能说明（函数、类）

本题采用单函数流程（`main`）实现，关键逻辑可概括为两步：

1. 建图：遍历输入，把父子关系改造成无向边。
2. BFS 分层扩散：每轮固定处理当前层节点数，用队列快照实现“按分钟计时”。

分层计时关键片段（原注释保持不变）：

```cpp
// 当前这一分钟有几个节点同时在向外放毒
int current_layer_size = q.size();
```

#### 2.2.5 调试分析（遇到的问题和解决方法）

问题与排查重点：
分钟数容易出现 +1 偏差。
原因：若每层循环都直接 `minutes++`，最后一层也会被计时。
修复：每轮结束后仅在队列非空时加 1，对应“还有新一轮感染”。

#### 2.2.6 总结和体会

这题最关键的转变是“化树为图”。只用树的单向层级无法表达逆向感染，必须切到图论视角。BFS 的分层模型天然对应“每分钟一层扩散”，因此思路清晰且实现稳定。

复杂度分析：
1. 建图：每个节点最多处理两条边，$O(n)$。
2. BFS：每个节点、每条边最多访问一次，$O(n + e)$，树转无向图后 $e \approx 2(n-1)$，故总体为 $O(n)$。
3. 空间复杂度：邻接表 + 队列 + visited，$O(n)$。

### 2.3 二叉树的同构

#### 2.3.1 问题描述

给定两棵二叉树，允许任意节点左右子树交换，判断两树是否同构，并输出两树深度。

#### 2.3.2 基本要求

1. 正确构建两棵树并找到根节点。
2. 正确判断同构（包含交换和不交换两种匹配）。
3. 输出 `Yes/No` 以及两树深度。

#### 2.3.3 数据结构设计

```cpp
struct BiTNode
{
    char data;
    int left;
    int right;
};

BiTNode BT1[10105], BT2[10105];
```

读取子节点下标的关键片段（原注释保持不变）：

```cpp
string lc, rc;

if (lc != "-")
{
    BT[i].left = stoi(lc);
    checkfather[stoi(lc)] = true; // 编号为lc的节点有父亲了
}
```

#### 2.3.4 功能说明（函数、类）

```cpp
int MakeBiTree(BiTNode BT[]);
```

功能说明：
1. 读取节点信息并填充数组表示。
2. 通过 `checkfather` 找到没有父节点的根。

```cpp
bool isIso(int R1, int R2);
```

功能说明：
1. 递归判断两棵子树是否同构。
2. 同时检查“不交换匹配”和“交换匹配”。

```cpp
int getDepth(int root, BiTNode BT[]);
```

功能说明：
1. 递归计算树高。
2. 空树深度为 0。

#### 2.3.5 调试分析（遇到的问题和解决方法）

本题最深刻的调试问题不是递归逻辑，而是输入流处理。

问题现象：
在 OJ 上出现 Runtime Error 和 Memory Limit Exceeded，且错误集中在大数据组。

排查过程：
1. 先验证 `isIso` 递归终止条件，未发现死递归。
2. 再排查输入链路，发现早期实现曾用 `char lc, rc` 读取孩子编号。
3. 当输入出现两位及以上编号（如 `12`）时，`char` 只能读入 `'1'`，`'2'` 残留在输入流中，导致后续全体错位。
4. 输入流错位后，数组下标和节点关系被污染，最终触发 RE/MLE。

修复方案：
1. 改用 `string lc, rc` 完整读取字段。
2. 对非 `"-"` 的字段使用 `stoi` 转整数。

结果：
修复后数据流恢复稳定，程序通过全部测试点。

#### 2.3.6 总结和体会

这题的递归定义并不复杂，真正的难点在“工程层面的输入可靠性”。算法正确只是必要条件，输入链路稳定才是通过大数据测试的充分条件。

复杂度分析：
1. 建树：$O(n)$。
2. 深度计算：$O(n)$。
3. 同构判断：实现中同时尝试交换/不交换分支，最坏可达 $O(n^2)$，在一般测试数据下表现接近线性。
4. 总体空间复杂度：$O(n)$（主要为数组与递归栈）。
