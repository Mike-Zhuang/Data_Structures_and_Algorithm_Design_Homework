## 1. 涉及数据结构和相关背景
本次实验主要围绕“查找”这一核心主题展开。查找（Search）是在大量信息中寻找满足特定条件的记录的过程，是计算机最基本、最常用的操作之一。
实验内容涵盖了三种经典结构与算法：
1. **基于数组的查找**：借助排序和前缀和将问题转化为有序序列，从而使用二分查找算法（Binary Search）将时间复杂度从 $O(n)$ 降至 $O(\log n)$。
2. **二叉排序树（Binary Search Tree, BST）**：一种动态查找表。BST的左子树所有节点值小于根节点，右子树所有节点值大于根节点。它支持快速查找、插入和删除操作。在引入计数器（Count）后，它还能有效处理重复元素。
3. **哈希表与树形关系的降维（Hash Map & Array representation of Tree）**：通过哈希表（`std::map`）将复杂的字符串信息映射为数字ID，利用一维数组记录各节点的父节点指针（并查集/向上追溯的思想），以线性的方式重构和保存多叉树拓扑结构，从而实现空间和时间上高效的树上关系查询。

## 2. 实验内容

### 2.1 和有限的最长子序列
#### 2.1.1 问题描述
给你一个长度为 $n$ 的整数数组 `nums` 和一个长度为 $m$ 的整数数组 `queries` ，返回一个长度为 $m$ 的数组 `answer` ，其中 `answer[i]` 是 `nums` 中元素之和小于等于 `queries[i]` 的子序列的最大长度。子序列是由一个数组删除某些元素但不改变其在原数组中的顺序得到的一个新数组。

#### 2.1.2 基本要求
要求输入数据数组，根据 `queries` 中的每一个限制值，求出能选出的最大元素数量。数据规模达到 $m,n \le 1000$，要求在时限内输出结果，因此对算法复杂度有一定要求。

#### 2.1.3 数据结构设计
本题只需使用一维数组即可完成计算，其中前缀和数组天然具有单调递增性。
```cpp
// 核心数组定义
int nums[n];       // 原数组
int queries[m];    // 查询限制数组
int nums_sum[n];   // 前缀和数组
int answer[m];     // 答案数组
```

#### 2.1.4 功能说明
核心通过先将数组从小到大排序，然后求出前缀和。利用二分查找加速计算过程。
```cpp
/**
 * @brief 求出满足前缀和小于等于 query[i] 的最大索引
 * @param nums_sum  单调递增的前缀和数组
 * @param queries   查询值
 * @param answer    存储每个查询的答案
 */
for (int i = 0; i < m; i++) {
    int left = 0, right = n - 1;
    answer[i] = 0;
    while (left <= right) {  // 二分查找区间
        int mid = (left + right) / 2;
        if (nums_sum[mid] <= queries[i]) {
            // 如果当前前缀和满足条件，则尝试向右扩大区间
            answer[i] = mid + 1; 
            left = mid + 1;
        } else {
            // 否则缩小所选元素个数（向左折半）
            right = mid - 1;
        }
    }
}
```

#### 2.1.5 调试分析
**从线性暴力的 break 陷阱到二分查找的降维打击**
- **踩坑**：一开始在处理“挑选尽量多的元素”时，采用了暴力双重循环（时间复杂度 $O(m \times n)$）线性查找前缀和数组。由于逻辑失误，在内层循环中，一旦发现 `nums_sum[j] <= queries[i]` 就立刻赋值并 `break`。这就导致程序只记录了“最小的满足条件的索引”，而不是最大长度，导致遇到样例数据输出了错误结果。且未对答案数组进行有效初始化，潜藏了内存越界风险。
- **破局**：彻底摒弃了顺序查找，转而利用前缀和数组天然单调递增的特性，引入了二分查找。每次遇到查询，直接折半缩小范围。这不仅避开了 `break` 提前截断的逻辑陷阱，还将单次查询的比较次数大幅降低，整体时间复杂度降至 $O(m \times \log n)$。

#### 2.1.6 总结和体会
处理贪心或者子序列和问题时，**排序+前缀和**是常用的预处理手段。而在有序数组上进行查找搜索，必须具备条件反射般使用**二分查找**的意识。理解边界条件（如 `left <= right` 和 `answer[i] = mid + 1;` 的更新时机）是写出正确二分算法的核心难点。

### 2.2 二叉排序树
#### 2.2.1 问题描述
本题要求实现一个维护整数集合（允许有重复关键字）的二叉排序树（BST），并支持五种操作：插入数值、删除数字（若有多个仅删除一个）、查询某个整数存在的个数、查询当前集合中的最小值、查询某个数字的前驱（集合中比该数字小的最大值）。

#### 2.2.2 基本要求
根据给定的多组指令列表（$n \le 1000$）准确进行数据结构的增、删、查、改操作并输出查询结果。

#### 2.2.3 数据结构设计
因为允许出现重复元素，传统BST会退化。在这里对节点引入了频率计数器 `count` 字段，避免构造“长链废树”。
```cpp
// Definition for a BST node.
struct TreeNode {
    int val;           // 节点存放的元素大小
    int count;         // 元素频率（重复出现次数）
    TreeNode *left;    // 左孩子指针
    TreeNode *right;   // 右孩子指针
    TreeNode(int v) : val(v), count(1), left(nullptr), right(nullptr) {}
};
```

#### 2.2.4 功能说明
对树的所有操作封装在了 `class BST` 当中。插入时物理上的合并可以缩减结构复杂度：
```cpp
/**
 * @brief 移除二叉树中的指定值
 * @param node     当前遍历的节点
 * @param x        需要被删除的值
 * @param deleted  标识是否成功删除的布尔引用
 * @return         返回调整后的子树根节点
 */
TreeNode *remove(TreeNode *node, int x, bool &deleted) {
    if (!node) return nullptr;
    if (x < node->val) { node->left = remove(node->left, x, deleted); }
    else if (x > node->val) { node->right = remove(node->right, x, deleted); }
    else { // 命中待删除节点
        deleted = true;
        if (node->count > 1) {
            node->count--; // 存在重复值，逻辑删除
        } else {
            // 叶子或单支节点直接返回其非空子节点并删除本身
            if (!node->left) { TreeNode *temp = node->right; delete node; return temp; }
            if (!node->right) { TreeNode *temp = node->left; delete node; return temp; }
            
            // 拥有双子树节点时，找到右子树中最小节点（后继节点）
            TreeNode *temp = findMinNode(node->right);
            node->val = temp->val;
            node->count = temp->count;
            temp->count = 1;
            node->right = remove(node->right, temp->val, deleted); // 递归删除后继节点
        }
    }
    return node;
}
```

#### 2.2.5 调试分析
**状态合并不造废树，双支节点劫持指针**
- **初始误区**：在面对大量重复元素时，原教旨主义BST可能会因为向单侧延伸而退化成一条 $O(n)$ 的单链表。且一开始直觉上误以为可以将较小的值强制挂载到右侧某个较大节点的左下方，后来意识到这违背了BST自顶向下严格路由的铁律。
- **优化**：在架构上摒弃了为重复元素分配新内存的做法，节点内部引入 `count` 进行“逻辑增减”，只有频次归零时才触发真实内存释放。在调试最复杂的“删除双子树节点”场景时，确立了“寻找右子树最小节点 -> 覆盖目标节点 -> 递归删除后继节点”的指针转移规范，有效避免了野指针和拓扑结构的断裂。

#### 2.2.6 总结和体会
二叉查找树的形态极度依赖插入顺序，引入 `count` 后很大程度上缓解了树深退化的问题。其次体会到了指针和引用在树结构变化中所占的核心地位，特别是双分支节点的删除其实是一次“值替换”配合“退化到单分支/叶子节点删除”的巧妙降维转换。

### 2.3 家族树
#### 2.3.1 问题描述
教授收集了通过缩进表示的文本树形文件（表示家谱）。需要实现一个程序，读取这种带空格缩进的文本格式构造家族树网络，并依据给定的关系查询语句（如判断 X 是否为 Y 的孩子/父母/兄弟/后代/祖先），对给出的判断语句输出 True 或 False。

#### 2.3.2 基本要求
输入由文本格式构成的多级树，利用算法将其转化为易搜索的数据结构。查询数量 $m$ 和人员规模 $n$ 都高达 $5000 \sim 10000$ 且存在多组测试样例。这要求避免全表暴力比对，对I/O解析及数据存取效率有很高要求。

#### 2.3.3 数据结构设计
抛弃了多叉树节点指针的传统设计，使用哈希表（`std::map`）和数组进行“数据降维”。数组 `parent_id` 记录直接父节点的ID，从而以完全扁平的结构表示了一棵多叉树。
```cpp
const int MAXN = 10010;
int parent_id[MAXN];     // parent_id[i] 代表 ID为i的节点的父节点ID
int depth_track[MAXN];   // depth_track[k] 记录由于缩进深度为 k 最近扫描到的节点ID
map<string, int> name_to_id; // 哈希表，将人名映射为唯一整数ID
```

#### 2.3.4 功能说明
通过追踪前导空格数量获得层级关系，利用数组进行线性的回溯查找祖先关系。
```cpp
/**
 * @brief 解析文本前导空格并确立父子关系
 * ... (代码读取与映射)
 */
if (space_count == 0) {  // 树根
    parent_id[i] = -1;  
} else {                 // 根据缩进量，父节点即为上一个缩进层级的节点
    parent_id[i] = depth_track[space_count - 1]; 
}
depth_track[space_count] = i; // 更新当前层级最近的节点

/**
 * @brief 查找两个节点s1(u)和s6(v)之间是否存在Descendant(后代)关系
 */
int current = u;
while (current != -1) {  // 不断溯祖，看能否碰到v
    if (current == v) {
        is_true = true;
        break;
    }
    current = parent_id[current];
}
```

#### 2.3.5 调试分析
**数据降维与 I/O 边界防御**
- **I/O踩坑**：如果试图使用 `cin` 混用 `getline`，`cin` 会遗留换行符，且会吃掉前导空格导致表示唯一亲属依赖关系的深度/缩进数完全丢失。且文本尾部的句号会被连同单词一起读入，导致哈希表 Key 不纯净。
- **清洗与破局**：第一步强制执行“清洗”，在获得输入后做 `s6.pop_back()` 抹除句号。第二步做“数据降维”，引入哈希表把名字转换为编号。再使用一维数组配合 `depth_tracker`，仅靠一次 $O(n)$ 线性扫描就把空格缩进量完美的转换为结构拓扑。在处理查询时利用整数 ID 执行 `curr = parent[curr]`，成功将图论转化为极简高效的 $O(H)$ 获取路径运算。

#### 2.3.6 总结和体会
最大的启示是：**不要死板地遵循直觉去建造复杂的指针多叉树**。很多时候，利用并查集思想（只记录每个节点的 `parent`），加上哈希映射，能把十分臃肿复杂的文本树极度拍平。这不仅降低了空间的浪费，更大大加快了寻址和回溯的效率，是极其精妙的数据降维打击。
